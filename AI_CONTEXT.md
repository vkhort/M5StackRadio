# AI Context: M5StackRadio (Classic Core / M5Unified)

## 🎯 Глобальная цель проекта
Продвинутый многопоточный интернет-радиоприемник на базе контроллера M5Stack Core (Basic/Gray/Fire) с физическими кнопками (ESP32, 240 МГц) на фреймворке Arduino с использованием FreeRTOS, файловой системы SPIFFS и экосистемы M5Unified.

## ⚙️ Аппаратная спецификация
*   **Плата:** M5Stack Core (Basic/Gray/Fire)
*   **Ядро:** ESP32 (240 МГц, двухъядерный)
*   **Экран:** ILI9341 (320x240, через `M5GFX`)
*   **Звук:** Встроенный динамик через усилитель NS4148 (I2S: BCLK=12, LRC=13, DIN=15)
*   **Кнопки:** `M5.BtnA` (Левая), `M5.BtnB` (Средняя), `M5.BtnC` (Правая) из экосистемы `M5Unified`.

## 🔄 Жизненный цикл и логика инициализации
При включении устройства строго соблюдается следующий алгоритм:

1.  **Монтирование FS**: Класс `WorkSPIFFS` монтирует память SPIFFS.
2.  **Проверка конфига**: Проверяется наличие текстового файла `/config` (формат: `ключ=значение`).
    *   **Если файла нет:** Метод `setDefaults()` генерирует его «на лету», записывая константы по умолчанию из `config.h` (SSID, Password, стартовая громкость, список станций).
    *   **Если файл есть:** Из него в оперативную память загружаются параметры текущей сессии (`ConfigData`) и динамический массив станций.
3.  **Инициализация Wi-Fi**: Класс `WiFiConnect` пытается подключить устройство к роутеру, используя SSID и пароль из файла `/config` (таймаут 15 секунд).
4.  **Разветвление режимов (STA / AP)**:
    *   **STA Режим (Успех)**: Если Wi-Fi подключен, запускается синхронизация времени по NTP, поднимается асинхронный веб-сервер и стартуют 3 параллельные задачи FreeRTOS. Радио начинает играть.
    *   **AP Режим (Отказоустойчивость)**: Если Wi-Fi не найден, M5Stack поднимает собственную точку доступа (SSID/PASS берутся из `/config`). Аудиопоток не воспроизводится. Устройство переходит в режим ожидания для перенастройки через браузер.

## 🏗 Архитектура задач FreeRTOS (в STA Режиме)
Межъядерное взаимодействие (Core 1 ➔ Core 0) осуществляется строго через потокобезопасные атомарные очереди `xQueue`.

*   **Core 0 (Аудиопоток)**: `AudioTask` (приоритет 5) — непрерывное декодирование MP3/HTTP потоков через библиотеку `ESP32-audioI2S` и заполнение I2S буфера. При наличии PSRAM сетевой буфер выделяется в ней.
*   **Core 1 (Интерфейсы и периферия)**:
    *   `ControlTask` (приоритет 2, 50 Гц) — опрос трех физических кнопок через `M5.update()` и обновление бегущей строки метаданных на экране.
    *   `NetworkTask` (приоритет 1) — обслуживание асинхронных запросов от веб-интерфейса браузера (`mathieucarbou/ESPAsyncWebServer`).

## 🎮 Логика управления (Кнопки)
*   `M5.BtnA` (Левая): Клик — уменьшить громкость (`CMD_VOLUME_DOWN`) / Удержание (600мс) — предыдущая станция (`CMD_STATION_PREV`).
*   `M5.BtnB` (Средняя): Клик — запуск/остановка потока (`CMD_TOGGLE_PLAY`).
*   `M5.BtnC` (Правая): Клик — увеличить громкость (`CMD_VOLUME_UP`) / Удержание (600мс) — следующая станция (`CMD_STATION_NEXT`).

## 📁 Структура проекта (Модульный принцип)
Проект построен по строгому модульному принципу. Перекрестное управление периферией из разных файлов запрещено:

1.  **`src/M5StackRadio.ino`**: Точка входа. Инициализирует глобальные объекты (`Display`, `WorkSPIFFS`, `WiFiConnect`, `Radio`). Настраивает `M5Unified`. Запускает логику сети и планировщик FreeRTOS. Системный цикл `loop()` разгружен (`vTaskDelay(10)`).
2.  **`lib/vk_lib/config.h`**: Хранит жесткие дефайны (геометрия экрана, тайминги, приоритеты, URL потоков по умолчанию).
3.  **`lib/vk_lib/display.h`, `display.cpp`**: **Единственная точка взаимодействия с экраном.** Инкапсулирует все графические команды. Задачи FreeRTOS взаимодействуют с экраном исключительно через публичные методы этого класса (`display.drawTime()`, `display.updateVolume()`).
4.  **`lib/vk_lib/radio.h`, `radio.cpp`**: Ядро системы. Разворачивает многопоточность через `xTaskCreatePinnedToCore`. Обрабатывает очереди `xQueue`. Внутри `ControlTask` обрабатываются физические кнопки.
5.  **`lib/vk_lib/classes.h`, `classes.cpp`**: Реализует классы `WorkSPIFFS` (менеджмент файла `/config`) и `WiFiConnect` (логика STA/AP подключения).
6.  **`platformio.ini`**: Настройки компилятора под плату `m5stack-core-esp32`. Включает оптимизацию, поддержку PSRAM и декларирует внешние зависимости.

## ⛔ Критические правила для ИИ (СОБЛЮДАТЬ СТРОГО!)
1.  **Правило ссылок**: Запрещено присылать ссылки на GitHub одной строкой. Любая ссылка должна разбиваться строго на ДВЕ строки.
    *   *Пример:* `https://github.com/` <перенос строки> `mathieucarbou/ESPAsyncWebServer.git`
2.  **Асинхронность**: Категорический запрет на использование блокирующего `delay()`. Все паузы внутри задач — только через `vTaskDelay` или `vTaskDelayUntil`.
3.  **Код интерфейса**: Использовать только современные методы `M5.update()`, `M5.Display` и `M5.BtnX` из экосистемы `M5Unified`.
4.  **Изоляция графики**: Запрещено писать методы отрисовки интерфейса (координаты, цвета, очистку зон) внутри задач FreeRTOS или главного скетча. Все изменения экрана — СТРОГО внутри методов класса `Display`. Задачи взаимодействуют с экраном только через публичный интерфейс класса `Display`.
